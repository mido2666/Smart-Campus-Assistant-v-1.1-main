# Professor User Guide - Secure Attendance System

## Overview

The Secure Attendance System provides professors with comprehensive tools to manage attendance sessions, monitor student attendance in real-time, detect fraud, and generate detailed reports. This guide covers all features and workflows available to professors.

## Getting Started

### 1. Login and Dashboard

#### Accessing the System
1. Navigate to the Secure Attendance System URL
2. Enter your professor credentials
3. You'll be redirected to the Professor Dashboard

#### Dashboard Overview
The dashboard provides:
- **Real-time Attendance Overview**: Live attendance statistics
- **Active Sessions**: Currently running attendance sessions
- **Fraud Alerts**: Security alerts and warnings
- **Security Metrics**: System security performance
- **Quick Actions**: Common tasks and shortcuts

### 2. Profile Setup

#### Account Configuration
1. Click on your profile icon in the top-right corner
2. Select "Settings" from the dropdown menu
3. Configure your preferences:
   - **Notification Settings**: Email and push notifications
   - **Security Preferences**: Fraud detection sensitivity
   - **Display Options**: Theme and layout preferences

## Session Management

### 1. Creating Attendance Sessions

#### Basic Session Creation
1. Navigate to "Sessions" in the main menu
2. Click "Create New Session"
3. Fill in the session details:
   - **Course**: Select from your courses
   - **Title**: Session title (e.g., "Lecture 1: Introduction")
   - **Description**: Optional session description
   - **Start Time**: Session start date and time
   - **End Time**: Session end date and time

#### Advanced Security Settings
Configure security options for enhanced fraud prevention:

**Location Settings:**
- **Require Location**: Students must be within specified radius
- **Location Radius**: Set allowed distance from session location
- **Location Name**: Descriptive location name

**Device Security:**
- **Require Device Check**: Verify student devices
- **Device Registration**: Students must register devices
- **Device Sharing Detection**: Detect shared devices

**Photo Verification:**
- **Require Photo**: Students must take attendance photos
- **Photo Quality**: Set minimum photo quality threshold
- **Face Detection**: Optional face verification

**Fraud Detection:**
- **Enable Fraud Detection**: Activate ML-based fraud detection
- **Risk Thresholds**: Set fraud detection sensitivity
- **Alert Notifications**: Configure fraud alert settings

#### Session Templates
Save frequently used session configurations:
1. Create a session with your preferred settings
2. Click "Save as Template"
3. Name your template (e.g., "Lecture Session")
4. Use templates for quick session creation

### 2. Managing Sessions

#### Starting Sessions
1. Navigate to "Sessions" → "Scheduled"
2. Find your session and click "Start Session"
3. Confirm session start
4. Share the QR code with students
5. Monitor real-time attendance

#### Session Monitoring
During active sessions, monitor:
- **Live Attendance Count**: Real-time attendance numbers
- **Student Status**: Present, absent, late, excused
- **Security Alerts**: Fraud detection warnings
- **Location Verification**: GPS validation status
- **Device Verification**: Device security status

#### Stopping Sessions
1. Click "Stop Session" on the active session
2. Confirm session end
3. Review final attendance summary
4. Generate attendance report

#### Emergency Stop
For urgent situations:
1. Click "Emergency Stop" on active session
2. Provide reason for emergency stop
3. System will immediately stop session
4. All students will be notified

### 3. Session Analytics

#### Real-time Analytics
Monitor session performance:
- **Attendance Rate**: Percentage of students present
- **Fraud Attempts**: Number of fraud attempts detected

### 2. Manual Attendance

#### Marking Attendance
Manually mark student attendance:
1. Navigate to "Students" → "Manual Attendance"
2. Select the session
3. Choose students to mark
4. Set attendance status:
   - **Present**: Student is present
   - **Absent**: Student is absent
   - **Late**: Student arrived late
   - **Excused**: Student has valid excuse
5. Add notes if needed
6. Save changes

#### Bulk Operations
Perform bulk attendance operations:
- **Mark All Present**: Mark all students present
- **Mark All Absent**: Mark all students absent
- **Excuse Students**: Mark students as excused
- **Export Attendance**: Export attendance data

## Troubleshooting

### 1. Common Issues

#### Session Issues
**Problem**: Session won't start
**Solution**: 
1. Check session configuration
2. Verify time settings
3. Ensure location is set
4. Contact support if needed

**Problem**: Students can't scan QR code
**Solution**:
1. Verify QR code is displayed
2. Check session is active
3. Ensure students have proper permissions
4. Test QR code functionality

#### Security Issues
**Problem**: False fraud alerts
**Solution**:
1. Review alert details
2. Check student location/device
3. Adjust fraud detection sensitivity
4. Contact security team if needed

**Problem**: Location verification failing
**Solution**:
1. Check GPS accuracy
2. Verify location radius
3. Ensure students have location permissions
4. Test location verification
