# Student User Guide - Secure Attendance System

## Overview

The Secure Attendance System provides students with a secure and user-friendly way to mark attendance. This guide covers all features and workflows available to students, including the multi-step security verification process.

## Getting Started

### 1. Login and Access

#### Accessing the System
1. Navigate to the Secure Attendance System URL
2. Enter your student credentials
3. You'll be redirected to the Student Dashboard

#### Dashboard Overview
The dashboard provides:
- **Active Sessions**: Currently available attendance sessions
- **Attendance History**: Your past attendance records
- **Security Status**: Your device and security information
- **Notifications**: Important announcements and alerts

### 2. Profile Setup

#### Account Configuration
1. Click on your profile icon in the top-right corner
2. Select "Settings" from the dropdown menu
3. Configure your preferences:
   - **Notification Settings**: Email and push notifications
   - **Privacy Settings**: Location and photo permissions
   - **Security Preferences**: Device registration and security
   - **Display Options**: Theme and layout preferences

## Marking Attendance

### 1. Finding Active Sessions

#### Available Sessions
1. Navigate to "Attendance" in the main menu
2. View available sessions:
   - **Session Title**: Course and session name
   - **Time**: Session start and end times
   - **Location**: Session location (if applicable)
   - **Requirements**: Security requirements (location, photo, device)

#### Session Requirements
Before marking attendance, check session requirements:
- **Location Required**: GPS location verification needed
- **Photo Required**: Attendance photo required
- **Device Check**: Device verification required
- **Time Window**: Allowed attendance time window

### 2. Multi-Step Attendance Process

#### Step 1: QR Code Scanning
1. Click "Mark Attendance" on an active session
2. Allow camera access when prompted
3. Point your camera at the QR code displayed by your professor
4. Wait for QR code verification
5. Verify the session information displayed

**Troubleshooting QR Code Issues:**
- **Camera not working**: Check camera permissions
- **QR code not scanning**: Ensure good lighting and steady hands
- **Invalid QR code**: Contact your professor
- **Session expired**: Check session time window

#### Step 2: Location Verification
If location is required:
1. Allow location access when prompted
2. Wait for GPS location to be determined
3. Verify your location is within the allowed radius
4. Check location accuracy (should be within 10 meters)

**Location Requirements:**
- **GPS Accuracy**: Location must be accurate within 10 meters
- **Distance**: Must be within session location radius
- **Permission**: Location permission must be granted
- **Network**: Internet connection required for location services

**Troubleshooting Location Issues:**
- **Location denied**: Grant location permission in browser settings
- **Poor accuracy**: Move to an area with better GPS signal
- **Outside radius**: Move closer to the session location
- **Location not found**: Check internet connection and try again

#### Step 3: Device Verification
If device check is required:
1. Your device will be automatically verified
2. Check that your device is registered
3. Verify device security status
4. Ensure device compliance

**Device Requirements:**
- **Device Registration**: Device must be registered
- **Security Check**: Device must pass security verification
- **Browser Compatibility**: Use supported browser
- **JavaScript**: JavaScript must be enabled

**Troubleshooting Device Issues:**
- **Device not registered**: Register your device first
- **Security check failed**: Contact support
- **Browser issues**: Use supported browser
- **JavaScript disabled**: Enable JavaScript

#### Step 4: Photo Capture
If photo is required:
1. Allow camera access when prompted
2. Position yourself in the camera frame
3. Ensure good lighting and clear visibility
4. Take the attendance photo
5. Verify photo quality and approval

**Photo Requirements:**
- **Quality**: Photo must meet minimum quality standards
- **Lighting**: Adequate lighting required
- **Clarity**: Face must be clearly visible
- **Format**: Photo must be in supported format

**Troubleshooting Photo Issues:**
- **Camera not working**: Check camera permissions
- **Poor quality**: Improve lighting and positioning
- **Photo rejected**: Retake photo with better quality
- **Format issues**: Use supported photo format

#### Step 5: Final Confirmation
1. Review all verification steps
2. Check security summary
3. Confirm attendance submission
4. Wait for confirmation

**Confirmation Process:**
- **Security Summary**: Review all security checks
- **Final Verification**: Confirm all requirements met
- **Submission**: Submit attendance record
- **Confirmation**: Receive attendance confirmation

### 3. Attendance Status

#### Status Types
Your attendance will be marked as:
- **Present**: Successfully marked attendance
- **Late**: Marked attendance after grace period
- **Absent**: Did not mark attendance
- **Excused**: Excused absence (requires professor approval)

#### Status Indicators
- **Green**: Present and on time
- **Yellow**: Late but present
- **Red**: Absent
- **Blue**: Excused absence
